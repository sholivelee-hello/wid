# 평평한 TASK 리스트 + 이슈 페이지 + 노션 동기화 규칙 — Design

날짜: 2026-06-03
출처: 사용자 직접 설계 + 비주얼 컴패니언 브레인스토밍 (목업: `.superpowers/brainstorm/10498-1780482772/content/`)
선행 스펙: `2026-06-03-ia-simplification-design.md` (메뉴 3개·시작화면 /today·보기 칩·상태 3값·우클릭 메뉴 — 모두 적용 완료 상태에서 출발)

## 사용자가 정의한 핵심 요구

1. 모든 TASK는 ISSUE와 상관없이 **독립적으로(평평하게)** 보인다.
2. 각 TASK 카드에 **소속 ISSUE를 대놓고** 표시한다.
3. **ISSUE별 모아보기**가 가능해야 한다 — 해당 ISSUE의 TASK 목록·순서·진행 정보를 한눈에 보는 페이지.
4. 노션発 TASK의 이름을 WID에서 바꿔도 **노션에는 반영되지 않는다**. 단 노션에서 완료 처리되면 **이름이 바뀌었어도(ID 매칭) WID도 자동 완료**된다.

브레인스토밍으로 확정된 세부 결정은 아래 각 섹션에. TASK/ISSUE/sub-TASK 용어와 데이터 모델(3-level)은 유지.

## 1. 전체(/inbox) — 평평한 리스트

- **ISSUE 그룹핑 제거**: 등록 뷰는 모든 TASK(+ ISSUE 미연결 task)를 단일 평면 리스트로. ISSUE 헤더·트리 들여쓰기 없음.
- **정렬: 등록 최신순** (created_at desc). 전체 화면에서 **드래그 reorder 제거** — 수동 순서는 이슈 상세 페이지의 역할.
- **sub-TASK는 평소 숨김**: 리스트에는 TASK만. 부모 카드에 `↳ sub N` 표시, 클릭하면 그 자리에서 펼침(접힘 기본). sub-TASK 자체는 평평한 리스트에 직접 등장하지 않는다.
- **카드 = 두 줄 구성** (목업 `card-issue-label-v3.html`):
  - 1행: 완료 동그라미(○) + 출처 아이콘 + 제목
  - 2행(메타): `📁 ISSUE 칩` · 마감일 · 요청자 · `↳ sub N` — 해당 정보가 없으면 그 항목 생략, 메타가 하나도 없으면 한 줄 카드
  - 기존 카드의 해(오늘) 버튼·상태 드롭다운·⋯ 더보기 버튼은 **제거** (우클릭 메뉴·상세 패널로 일원화). 완료 동그라미만 상시 노출(평소 흐리게, hover 시 또렷).
- **출처 아이콘 = 실제 브랜드 로고, 표시 전용**:
  - 슬랙 = 공식 4색 로고 SVG, 노션 = 흰 바탕 + 검정 N, WID 직접 입력 = 키컬러 점(`bg-primary`)
  - 아이콘 클릭 동작 없음 (hover 열기 기능 만들지 않기로 확정)
  - 브랜드 컬러는 "새 색 금지" 원칙의 예외로 간주 — 출처 식별용 브랜드 아이콘에 한정 (CLAUDE.md 디자인 섹션에 기록할 것)
  - source enum에 **`jira` 슬롯 예약** (아이콘·연동은 별도 스펙, 이번 구현 범위 외)
- **우클릭 메뉴 맨 위 = "원본 열기"** (slack_url/notion 페이지가 있는 task만 노출, 새 탭).
- **ISSUE 칩 클릭 → 해당 이슈 상세 페이지로 이동.**
- **본문 폭**: 콘텐츠 컬럼 `max-width 720px` 중앙 정렬 (오늘/전체/이슈/돌아보기 공통).
- 보류·완료·휴지통 칩 뷰는 기존 유지하되 카드 스타일만 새 두 줄 카드로 통일.

## 2. 사이드바 — 메뉴 4개

오늘 · 전체 · **이슈**(신규) · 돌아보기 + 하단 설정 톱니. 이슈 메뉴 아이콘은 lucide `Folder` 계열.

## 3. 이슈 목록 페이지 (`/issues`, 신규)

목업 `issue-list.html`의 B안 (컴팩트 리스트):

- 한 줄에 이슈 하나: `📁 이름` + 미니 진행바(키컬러) + `완료 n/m`(tabular-nums) + 임박 마감(있으면, 키컬러 톤).
- 진행률 = 그 이슈 직속+하위 TASK 중 완료 비율 (취소는 분모에서 제외).
- **완료된 이슈**(모든 task 종결)는 목록 아래로 가라앉고 흐리게(opacity).
- 행 클릭 → 이슈 상세. 정렬: 진행 중 이슈 우선, 그 안에서 임박 마감순 → 최신순.

## 4. 이슈 상세 페이지 (`/issues/[id]`, 전면 개편)

목업 `issue-detail.html`:

- 헤더: 이슈 이름(h1) + `n/m 완료` + 큰 진행바.
- 그 아래 한 줄: **"다음: {첫 번째 미완료 task} · 마감 {date}"** — 이 이슈에서 지금 할 일을 앱이 지목.
- **진행 중 목록**: 사용자가 정한 순서(position) 그대로, **드래그(⠿) reorder는 여기서만 유지**. 카드 한 줄(grip + ○ + 출처 아이콘 + 제목 + sub N + 마감).
- `+ 이 이슈에 task 추가` 인라인 입력(생성 시 해당 issue로 연결, position 맨 아래).
- **완료/취소 목록**: 아래에 흐리게(취소선).
- 기존 `/issues/[id]`의 다른 기능은 새 레이아웃에 맞게 정리(필요한 것만 이식).

## 5. 노션 동기화 규칙

현재 sync(`src/app/api/notion/sync/route.ts`)는 매 동기화마다 노션 제목으로 WID 이름을 덮어씀 → **사용자 요구와 정반대, 수정 필수**.

- **이름 보호 플래그**: tasks에 `name_locked`(boolean, default false) 추가 — 마이그레이션 008.
  - 사용자가 WID UI에서 **노션発 task의 이름을 수정**하면 `name_locked = true` (인라인 에디터·상세 패널 공통, source가 notion일 때만 세팅).
  - sync는 `name_locked = false`인 task만 노션 제목 변경을 따라감. `true`면 영구히 WID 이름 유지.
- **WID → 노션 역반영 없음**: 현행 유지 (push 코드 자체가 없음 — 새로 만들지 않는다).
- **완료 동기화**: `notion_task_id`로 매칭, 노션 status가 Complete 그룹이면 WID `완료` — 이름과 무관. 현행 동작 유지·테스트로 보장.

## 6. 영향 범위 / 정리되는 것

- **dnd**: 인박스 4-context sortable(`docs/architecture/dnd.md`) 중 인박스 쪽 제거, 이슈 상세 1-context만 유지. 계층 이동 불가 invariant는 그대로.
- **오늘(/today) 화면**: 동일한 두 줄 카드 스타일 적용. 평평한 리스트(오늘은 이미 사실상 평면) + sub 접힘 동일.
- **hierarchyLabel/트리 렌더(`inbox-tree.tsx`, `task-branch.tsx`)**: 전체 화면에서는 미사용으로 전환, 이슈 상세·sub 펼침에서만.
- **문서 갱신**: `docs/architecture/hierarchy.md`(표시 정책), `dnd.md`, `inline-editing.md`(name_locked 세팅), `today.md`(카드 스타일), 신규 `issues.md`(이슈 페이지 계약), CLAUDE.md 디자인 섹션(브랜드 아이콘 예외, 본문 폭 720px, 메뉴 4개).

## 범위 외 (이번 구현에 포함하지 않음)

- JIRA 연동 (source enum 슬롯만 예약, 별도 스펙으로)
- 보류/휴지통 뷰의 기능 변경 (카드 스타일 통일만)
- 노션 쪽으로의 어떤 쓰기(push)도 하지 않음
