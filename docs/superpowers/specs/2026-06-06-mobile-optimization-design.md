# 모바일 최적화 전면 개편 — 설계 (2026-06-06)

## 배경

- 모바일 UX 전문 평가(Opus 에이전트, 2026-06-06) 결과 **38/100**. "읽기는 되지만 조작이 안 되는" 데스크톱-우선 앱 상태.
- 사용자 직접 보고 버그: **task list 화면(/today, /inbox, /issues/[id])에서 터치 스크롤이 안 됨.** 다른 화면은 정상.
  - 원인 확정: sortable task 줄 전체에 `touchAction: 'none'`이 걸려 있어 브라우저 터치 스크롤을 차단 (`src/components/inbox/inbox-tree.tsx:154`, `src/components/tasks/task-branch.tsx:111`).
- 사용 시나리오(사용자 인터뷰): 폰에서는 **확인 + 가벼운 처리**(오늘 확인, 완료 체크, 빠른 추가, 간단 분류)가 중심. 긴 편집·정렬은 데스크톱이 주력.

## 1단계 — "폰으로 쓸 수는 있게" (먼저 배포)

### ① 스크롤 버그 수정 (0순위)

- `touchAction: 'none'`을 sortable 줄 전체에서 제거하고 **드래그 활성화 요소(grip handle)에만** 적용 — dnd-kit 권장 패턴. 데스크톱 마우스 드래그는 그대로, 터치 스크롤 복원.
- 터치 기기(마우스 없음)에서는 grip을 아예 숨기고 드래그 센서를 비활성화한다. 적용 대상: `/today`, `/inbox`, `/issues/[id]`, `inbox-tree`의 4개 DndContext.

### ② 터치 타겟 최소 44px

- 시각 크기는 디자인 시스템 v3 그대로 두고, **히트 영역만** 투명 패딩/가상 영역으로 44×44px 이상 확보.
- 대상: 완료 토글 동그라미(task-card), 헤더 아이콘 버튼, /inbox 보기 칩, 상세 패널 내 소형 버튼(`h-6` 류), chevron(expand).

### ③ "⋯ 더보기" 버튼

- task 줄 우측에 점 3개 버튼 추가. 기존 `ContextMenuContent`의 항목(이슈 연결·상태 변경·보류·삭제·원본 열기)을 DropdownMenu로 재사용 — 우클릭 메뉴는 데스크톱에 그대로 유지.
- 노출: 마우스 기기 = hover 시(현행 액션들과 동일 언어), 터치 기기 = 항상(저채도).

## 2단계 — "폰에서 제대로 좋게"

### ④ 하단 탭바 + FAB

- 사이드바가 숨는 폭(`lg` 미만)에서 하단 고정 탭바: 오늘 · 전체 · 이슈 · 돌아보기. 활성 = 키컬러 아이콘+라벨(사이드바와 동일 언어), 비활성 = muted.
- 우하단 **키컬러 FAB** = 새 task(QuickCapture 모달). 헤더의 "새 task"·펼치기/접기 버튼은 폰에서 숨김 → 모바일 헤더 = 제목 + 동기화 + 설정 톱니.
- 햄버거 ☰ Sheet nav는 탭바 도입과 함께 폰에서 제거.
- `env(safe-area-inset-bottom)` 처리(아이폰 홈바). 본문 하단 패딩으로 탭바·FAB에 콘텐츠 가림 방지.

### ⑤ task 상세 바텀시트

- `sm`(640px) 미만에서 `task-detail-panel`을 shadcn **Drawer**(vaul) 기반 바텀시트로: 높이 ~90vh, 상단 손잡이, 쓸어내려 닫기.
- 키보드 대응: 포커스된 입력칸 scrollIntoView, 푸터 액션(저장·완료 등)은 시트 하단 고정, safe-area 패딩.
- 데스크톱(`sm` 이상)은 현행 중앙 Dialog 유지 — 시각 변경 없음.

### ⑥ 스와이프 액션

- 진행 중 task 목록에서: **왼쪽 스와이프 = 완료**(토스트 + 되돌리기), **오른쪽 스와이프 = 보류**(pend).
- 직접 구현(신규 제스처 라이브러리 없음): 가로 이동이 세로보다 명확히 클 때만 발동(스크롤 오인 방지), 임계 거리 미달 시 원위치 스냅백, 발동 전 색·아이콘 힌트 노출.
- 기존 PATCH API 재사용. 실패 시 토스트 알림 + UI 원복.
- 적용 범위: 진행 중 목록(오늘·전체·이슈 상세). 완료/휴지통/보류 칩 뷰는 이번 범위 밖.

### ⑦ 돌아보기 모바일 뷰

- `lg` 미만: 7열 미니 달력(날짜 셀에 task·일정 밀도를 키컬러 점으로) + 날짜 탭 시 아래 "그날 상세" 패널(완료 task·일정 목록).
- 데스크톱(`lg` 이상): 현행 월 캘린더 + 우측 패널 2단 유지.

## 기술 결정

- **분기 기준 이원화**:
  - 레이아웃(탭바·미니 달력) = 뷰포트 폭 `lg`(사이드바 숨김 지점과 일치), 바텀시트 = `sm`.
  - 인터랙션(grip 숨김·⋯ 상시 노출·스와이프·드래그 비활성) = **포인터 능력** (`(hover: none)`/`pointer: coarse`, Tailwind v4 `pointer-coarse:`/`pointer-fine:` variant 또는 matchMedia 훅). 폰 가로 모드에서도 터치 동작 유지, 아이패드+마우스는 데스크톱 동작.
- **신규 의존성 1개**: shadcn Drawer(vaul) — 바텀시트 쓸어내려 닫기. shadcn 우선 규칙 부합. README/CLAUDE.md 반영.
- 색·타이포·표면은 디자인 시스템 v3 토큰만 사용 — 새 색 없음.
