# IA 단순화 + 사용성 개선 (UX 평가 후속) — Design

날짜: 2026-06-03
배경: UX 평가 에이전트의 전면 평가 결과, "관공서 느낌"의 근본 원인을 (1) 분류 노동 전가, (2) 도구 우선 레이아웃, (3) 과다한 메뉴로 진단. 사용자가 방향 승인.

## 사용자 결정 (변경 금지)

- **TASK / ISSUE / sub-TASK 용어는 그대로 유지** — 용어 치환(M2)은 하지 않는다.
- 사이드바 키컬러 기둥(브랜드 표면)은 유지하되 가독성만 개선.

## 결정 사항

### 1. 시작 화면 = 오늘 (`/today`)
- 루트 `/`는 `/today`로 redirect (server component `redirect()`).
- 기존 인박스 페이지는 `/inbox`로 이동 (`src/app/page.tsx` → `src/app/inbox/page.tsx`).
- `/`를 가리키던 내부 링크(로고 등)는 `/today` 또는 `/inbox` 중 의미에 맞게 갱신.

### 2. 사이드바: 메뉴 3개 + 하단 설정 아이콘
- 메뉴: **오늘**(`/today`, Sun) · **전체**(`/inbox`, Inbox) · **돌아보기**(`/history`, History). separator 제거.
- **설정**: 사이드바 맨 아래 톱니바퀴 아이콘만 (`/settings`). 글자 메뉴에서 제거.
- **보류함 / 휴지통 메뉴 제거** → 전체 페이지 안의 보기 칩으로 흡수 (결정 3).
- 카운트 뱃지는 '전체' 항목에 표시 (기존 '인박스' 뱃지 이전).
- 비활성 글자 대비 상향: `text-primary-foreground/70` → `/85` (WCAG 대비 개선).

### 3. 전체(`/inbox`) 상단 보기 칩
- 칩 한 줄: **진행 중**(기본) · **보류** · **완료** · **휴지통**.
- `보류` 칩 = 기존 `/pending` 페이지 내용, `휴지통` 칩 = 기존 `/tasks/trash` 페이지 내용을 뷰로 흡수 (해당 페이지의 컴포넌트/로직 재사용, 복원·영구삭제 동작 유지).
- `완료` 칩 = 기존 "완료한 것도 보기" 토글을 대체 (완료된 것만 보기).
- 휴지통 칩은 맨 끝 + 시각적으로 살짝 구분 (실수 방지).
- 기존 라우트 `/pending`, `/tasks/trash`는 `/inbox?view=pending`, `/inbox?view=trash`로 redirect (북마크 호환).
- 보류함 invariant(`docs/architecture/pending.md`)는 그대로 — 표시 위치만 이동.

### 4. 오늘 = explicit ∪ 마감 임박 자동 포함
- effective today set 확장: 사용자가 해 아이콘으로 고른 것(explicit) ∪ **마감일이 오늘이거나 지난 미완료 task** (자동).
- 자동 포함된 항목은 구분 표시(예: "마감" 뱃지)로 왜 여기 있는지 자명하게.
- 자동 포함 항목의 해 토글 동작은 기존 explicit 모델과 충돌하지 않게 — `docs/architecture/today.md`의 explicit/effective 모델 문서를 함께 갱신.

### 5. 완료 피드백 강화
- task 완료 시 토스트에 오늘 누적 표시: "오늘 N개 완료" (today set 기준).

### 6. 인박스 도구바 기본 접힘
- 검색·필터·정렬 sticky 도구바를 기본 접힘으로 — 버튼 하나(돋보기/필터 아이콘)로 펼침.
- 펼침 상태는 localStorage로 기억하지 않음(매번 접힘 = 기본은 "일 목록").
- ⌘K는 접힘 상태에서도 동작 (누르면 펼치고 포커스).

## 적용하지 않는 것 (이번 범위 외)

- 용어 치환 (사용자가 명시적으로 거부)
- 계층 UI 비가시화 (M1) — 추후 별도 논의
- 카드 액션 다이어트, 상태 자동화 (M3, m1) — 추후
- 검색 일원화 (M5) — 추후

## 문서 갱신 의무

- `docs/architecture/today.md` — effective set 확장 반영
- `docs/architecture/pending.md` — 보류함 진입점 변경 반영
- `CLAUDE.md` 디자인 시스템 v3 레이아웃 줄 — 시작 화면/메뉴 구조 변경 반영
